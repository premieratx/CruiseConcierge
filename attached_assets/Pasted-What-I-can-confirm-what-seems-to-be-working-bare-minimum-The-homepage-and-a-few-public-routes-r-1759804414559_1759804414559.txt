How to test whether Google is seeing your content

Here are steps you (or your developer) can do to verify:

Google Search Console – URL Inspection

Input a specific service page (e.g. /atx-disco-cruise) and check “View Crawled Page / HTML”. See if the FAQ content or service paragraphs appear in the source.

Use “Test Live URL” to force fetch + render.

Fetch as Google / Mobile-Friendly Test

Use the Google Mobile-Friendly Test tool. It will show you what Google sees after rendering. If major sections are missing there, that’s a red flag.

Fetch the raw HTML (curl)

curl or “view source” in the browser before JavaScript runs. See what text is present.

Lighthouse / PageSpeed Insights

Run on critical pages. Look at metrics like First Contentful Paint, Time to Interactive, Largest Contentful Paint, Layout Shift, etc.

Check the sitemap / robots.txt

Ensure your dynamic routes are included in the sitemap. Check robots.txt for disallowed rules.

Check for structured data (in source HTML)

View page source and search for <script type="application/ld+json">. Do you see FAQPage, WebPage, etc.?

Recommendations: What you and your dev should do next

Here’s a prioritized plan to fix visibility and SEO:

1. Make key content server-rendered / pre-rendered

For each service route (e.g. /private-cruises, /atx-disco-cruise, etc.), render the full HTML (text, FAQ, headings) on the server or via static generation. Never rely on client-side JS alone for content that Google must index.

If you use React/Vite SPA, add a prerender step or a headless renderer that outputs static HTML for known routes.

2. Add structured data (FAQPage, etc.) in the HTML

As soon as the page loads (in HTML), include the FAQPage JSON-LD block with all Q&A pairs.

Also include WebPage schema, Organization schema, BreadcrumbList, etc.

3. Optimize performance / bundle size

Code-split your JS so pages only load what they need.

Use lazy loading / loading="lazy" for images not in view.

Compress and optimize images.

Minimize render-blocking CSS/JS.

Use caching, CDN, HTTP/2, etc.

4. Ensure every route is crawlable

Create a static sitemap including all service pages.

In your navigation and footers, include direct (HTML) links to all key pages (so crawlers can see them without JS).

Avoid hiding routes behind menus that require JS to expand.

5. Verify meta tags / unique titles, descriptions

Each page should have its own <title> and <meta name="description">.

Use headings (H1, H2) meaningfully; don’t duplicate across pages too heavily.

6. Test & monitor

After changes, re-submit your sitemap in Google Search Console.

Use URL Inspection to verify the rendered content is visible.

Monitor indexing, traffic, and whether FAQ rich snippets begin to appear.
