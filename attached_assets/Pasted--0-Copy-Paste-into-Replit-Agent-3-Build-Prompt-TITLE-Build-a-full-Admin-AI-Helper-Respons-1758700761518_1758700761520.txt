## D. Orchestrator (put in `src/agent/index.ts`)

* Initialize OpenAI client (Responses API) with model `o3-mini`.
* Provide `tools` array using the JSON Schemas from Section C.
* Accept chat turns: `{sessionId, messages[], userId}`.
* Maintain `previous_response_id` per session in `agent_sessions` to preserve short‑term context.
* Loop tool calls: on tool output event → map to local tool → run with `{dryRun:true}` when risky → feed `tool_result` back into Responses API until `completed`.
* Stream tokens to client via SSE.

**Event loop pseudocode:**

```ts
const resp = await openai.responses.create({
  model: 'o3-mini',
  reasoning: { effort: 'medium' },
  input: [systemMsg, devMsg, ...userMsgs],
  tools,
  tool_choice: 'auto',
  previous_response_id: session.prevId,
});

for await (const event of openai.responses.stream(resp)) {
  if (event.type === 'tool_call') {
    const tool = registry[event.tool.name];
    const args = validate(event.tool.arguments);
    const guardedArgs = await guard(tool, args, userId);
    const result = await tool.run(guardedArgs, ctx);
    await streamToolResult(result);
  }
}
```

## E. Memory & Knowledge (RAG)

* **Embeddings:** use `text-embedding-3-large`.
* **Chunking:** 1–2k tokens with overlap; store `doc_id, url, tags, chunk, embedding` in `kb_chunks`.
* **Retrieval:** cosine similarity in pgvector; top‑k with Maximal Marginal Relevance; add as `input` items to Responses API.
* **Auto‑ingest:** nightly crawls of docs/sites; tool‑driven on demand via `kb.ingestUrl`.
* **Session memory:** persist `previous_response_id`; back‑fill with a short recap summary when sessions get long.

## F. Security, Approvals, and Audit

* **RBAC:** roles: `owner`, `admin`, `editor`, `viewer`. Map tools to min role.
* **Approvals:** For `requiresApproval`, agent must emit a plan + preview diff. UI shows **Approve / Cancel**.
* **Guards:** rate limit (per user + global), token/time caps, dry‑run first for writes, disallow wildcards in `fs.write` paths, block secrets.
* **Audit:** log user, sessionId, tool, args (redacted), result, diffs.

# 1) Server code scaffolds (snippets)

### `src/agent/index.ts` (skeleton)

```ts
import OpenAI from 'openai';
import { registry, toolsArray } from './registry';
import { getSession, saveSession } from './memory/sessionStore';
import { guard } from './security/guard';

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY! });

export async function handleChat({ sessionId, messages, userId }) {
  const session = await getSession(sessionId);
  const base = {
    model: 'o3-mini',
    reasoning: { effort: 'medium' },
    input: messages,
    tools: toolsArray,
    tool_choice: 'auto',
    previous_response_id: session?.prevId,
  } as const;

  const stream = await openai.responses.stream(base);

  for await (const event of stream) {
    if (event.type === 'tool_call') {
      const { name, arguments: args } = event;
      const tool = registry[name];
      const safeArgs = await guard(tool, args, userId);
      const result = await tool.run(safeArgs, { userId, sessionId });
      await stream.send({ type: 'tool_result', tool_name: name, result });
    }
    if (event.type === 'response.completed') {
      await saveSession(sessionId, event.response.id);
    }
  }
}
```
