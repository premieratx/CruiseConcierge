What Was Fixed:

The main issue was that when URL parameters were loaded, the selectedDuration wasn't being set for weekday dates. Since October 28, 2025 is a Tuesday (weekday), the system requires a duration selection (3 or 4 hours). Without this, the availability API didn't know what duration to filter for, resulting in no time slots being displayed.
